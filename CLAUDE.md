# Orthodox Calendar — CLAUDE.md

## Project Overview

Multi-locale Orthodox Church calendar iOS app (Serbian + Russian + English). No API backend — fully self-contained with bundled JSON data generated by a Python pipeline.

## Architecture

```
OrthodoxCalendar/
├── App/                    # App entry, ViewModel
├── Models/                 # CalendarDay, CalendarFile, AppLanguage, AppTheme
├── Views/                  # SwiftUI views
├── Localization/           # sr.json, ru.json, en.json, calendar_{sr,ru,en}_2026.json
├── Engine/                 # JulianConverter, PaschaCalculator
├── Resources/              # Assets.xcassets, splash_logo.png
scripts/
├── shared/                 # Paschalion, fasting engine, lectionary engine, saint parser, build pipeline
├── serbian/                # Scrapers for pravoslavno.rs, crkvenikalendar.rs
├── russian/                # Scrapers for azbyka.ru, days.pravoslavie.ru
data/
├── processed/              # Intermediate JSON data per locale
├── output/                 # Final calendar JSONs
```

## Build & Run

```bash
# Generate Xcode project (requires xcodegen: brew install xcodegen)
xcodegen generate

# Build for simulator
xcodebuild -project OrthodoxCalendar.xcodeproj -scheme OrthodoxCalendar \
  -destination 'platform=iOS Simulator,name=iPhone 16,OS=18.6' build

# Regenerate calendar data
python3 scripts/shared/build_database.py
cp data/output/calendar_sr_2026.json OrthodoxCalendar/Localization/
cp data/output/calendar_ru_2026.json OrthodoxCalendar/Localization/
cp data/output/calendar_en_2026.json OrthodoxCalendar/Localization/
```

## Key Design Decisions

### No API Backend
The app was originally built on orthocal.info API but this caused endless issues (New Calendar vs Old Calendar dates, English text leaking, cache versioning). Removed entirely. All data comes from bundled JSON files.

### Data Pipeline
- **Saints**: Scraped from pravoslavno.rs (Serbian) and azbyka.ru (Russian), mapped by Julian date (fixed) + pascha distance (moveable)
- **Readings**: Computed by Typikon lectionary engine (ported from orthocal Go code), matched to scraped Bible text
- **Fasting**: 7-level algorithmic engine based on Typikon rules
- **Reflections**: St. Theophan (Russian) from days.pravoslavie.ru

### Lectionary Engine
Full Typikon algorithm including Lukan Jump. Computes correct readings for ANY year without scraping. Key formula: `LucanJump = 168 - (Sunday after Elevation of Cross)`. Readings SQL data from orthocal project.

### Language Switching
Critical SwiftUI gotcha: `LazyVStack` caches cells by ID. When switching locales, `CalendarDay.id` (gregorianDate) is the same for both languages. Fix: `.id(viewModel.loadedLocale)` on the LazyVStack forces complete rebuild. Also requires proper `Equatable` on CalendarDay and all sub-types.

### Fasting (Serbian vs Russian)
Serbian church does NOT distinguish fishRoe (икра) from fish — both show as "риба". Russian keeps the distinction. Configured in fasting_engine.py.

### CalendarDay Model
- `dayOfWeek`: Python convention (0=Mon, 6=Sun), NOT iOS convention
- `weekdayIndex`: computed property converting to localization array index (0=Sun)
- `isSunday`: checks `dayOfWeek == 6`
- `ScriptureReading.book`: optional (lectionary uses title instead)
- `ScriptureReading.reference`: optional

### Readings Order
Readings must appear in liturgical service order: Passion Gospels (21-32) → Hours (100-400) → Vespers (500) → Matins Gospel (700) → Epistle (800) → Gospel (900). Sorting done in generate_readings.py.

## Locales

Serbian (`sr`), Russian (`ru`), and English (`en`). `AppLanguage` enum has three cases. All switch statements must be exhaustive with `.sr`, `.ru`, and `.en`. English uses engine-only readings (no scraped Bible text).

## Data Files

| File | Purpose |
|------|---------|
| `sr.json` | Serbian UI labels, month names, feast name translations |
| `ru.json` | Russian UI labels, month names, feast name translations |
| `en.json` | English UI labels, month names, feast name translations |
| `calendar_sr_2026.json` | Complete Serbian calendar (saints, readings, fasting, 365 days) |
| `calendar_ru_2026.json` | Complete Russian calendar (saints, readings, fasting, 365 days) |
| `calendar_en_2026.json` | Complete English calendar (saints, readings, fasting, 365 days) |
| `splash_logo.png` | App splash screen image (loaded from bundle, NOT asset catalog) |

## Common Pitfalls

1. **Bundle resources**: Files in `data/output/` are NOT automatically in the app bundle. Must copy to `OrthodoxCalendar/Localization/` and list in `project.yml` resources.
2. **Asset catalog images**: May not load on physical devices. Use `Bundle.main.url(forResource:)` for reliability.
3. **Zachalo numbers**: Differ between OCA, Serbian, and Russian traditions. Match readings by chapter:verse reference, never by zachalo number.
4. **Julian dates**: All fixed feasts use Julian calendar dates. Gregorian = Julian + 13 days (for 1900-2099).
5. **Pascha distance**: Negative = before Pascha, positive = after. 0 = Pascha day.
6. **ForEach identity**: Never use gregorianDate alone as SwiftUI ID when locale can change — content differs but ID stays same.
