## Project Overview

Orthodox Church calendar with daily saints, scripture readings, fasting guidance, and saint biographies. Supports Serbian, Russian, and English calendars — fully offline. iOS app built with SwiftUI, bundled JSON data generated by a Python pipeline. Cloudflare Worker API serves the same data from R2. Android port lives at [Ludikure/OrthodoxCalendarAndroid](https://github.com/Ludikure/OrthodoxCalendarAndroid).

## Build & Run

```bash
# Generate Xcode project (requires xcodegen)
xcodegen generate

# Build iOS app
xcodebuild -project OrthodoxCalendar.xcodeproj -scheme OrthodoxCalendar \
  -destination 'platform=iOS Simulator,name=iPhone 16,OS=18.6' build

# Regenerate ALL calendar data (4 locales × 7 years = 28 files)
python3 scripts/shared/build_database.py

# Or generate specific year range
python3 scripts/shared/build_database.py 2026 2026

# Copy to app bundle + compact (strip nulls, whitespace)
cp data/output/calendar_*.json OrthodoxCalendar/Localization/

# Deploy Cloudflare Worker
cd worker && npm install && wrangler deploy

# Upload data to R2
for f in data/output/calendar_*.json; do
  wrangler r2 object put "orthodox-calendar-data/$(basename $f)" --file "$f" --content-type "application/json" --remote
done

# Android build (separate repo)
cd ~/Developer/OrthodoxCalendarAndroid && ./gradlew assembleDebug
```

## Locales

Four locales with `AppLanguage` enum: `.sr`, `.ru`, `.en`, `.en_nc`.

| Locale | Calendar | Fixed Feasts | Fasting Rules |
|--------|----------|-------------|---------------|
| `sr` | Old (Julian) | Jan 7 = Christmas | SPC rules (stricter Wed/Fri) |
| `ru` | Old (Julian) | Jan 7 = Christmas | RPC rules |
| `en` | Old (Julian) | Jan 7 = Christmas | OCA/Russian rules |
| `en_nc` | Revised Julian | Dec 25 = Christmas | OCA rules |

`en_nc` shares `en.json` localization file but loads `calendar_en_nc_*.json` data. All switch statements use `case .en, .en_nc:` for English text.

## Data Pipeline

Flow: **Scrapers → Processed JSON → Build Pipeline → Calendar JSONs**

### build_database.py
- Generates `calendar_{locale}_{year}.json` for each locale/year
- Loads saints (by Julian date), readings (lectionary engine + scraped text), fasting (algorithmic)
- Injects moveable feasts algorithmically from `MOVEABLE_FEASTS` dict (keyed by pascha distance)
- Injects fixed great feasts from `FIXED_GREAT_FEASTS` dict (keyed by Julian date)
- For `en_nc`: uses Gregorian dates for fixed feasts (`NC_GREAT_FEASTS_GREG`)
- Feast descriptions loaded from `data/processed/shared/feast_descriptions.json` (separate file to avoid Python string quoting issues with Cyrillic quotation marks)
- SPC-specific fasting rules: stricter regular Wed/Fri, different Nativity/Apostles' Fast, Lazarus Saturday = oil not fish roe
- Bold saint feast rank upgrades fasting to oil during fasting periods (SPC)

### Key: Moveable vs Fixed
- **Moveable feasts** (Pascha, Holy Week, Pentecost): `moveable: true` flag in JSON. Injected by pascha distance. Year-dependent.
- **Fixed feasts** (Christmas, Theophany): Keyed by Julian month-day. Same Gregorian date every year (Julian+13).
- Saints data scraped for 2026 — moveable entries stripped and replaced algorithmically for all years.
- Saint bios are year-independent (keyed by date, not year).

### Fasting Engine
7 levels: `totalAbstinence`, `dryEating`, `hotNoOil`, `hotWithOil`, `fish`, `fishRoe`, `free`. Locale-aware (`compute_fasting(..., locale)`) with SPC overrides for Serbian.

## Key Design Decisions

### No API Backend
All data bundled as JSON. The Cloudflare Worker + R2 serves the same data for future on-demand loading but is not required — app works fully offline.

### CalendarDay Model
- `dayOfWeek`: Python convention (0=Mon, 6=Sun), NOT iOS convention
- `weekdayIndex`: computed property converting to localization array index (0=Sun)
- `Feast.moveable`: distinguishes algorithmic feasts from fixed saints (used for bio matching)
- `Feast.description`: expandable text for major feasts (from feast_descriptions.json)
- `FastingInfo.abbrev` and `.icon`: optional (stripped during compaction to reduce bundle size)

### Saint Bio Matching (DayDetailView.findBio)
Each saint card can expand to show a biography. Matching logic:
- `commonWords` filter excludes generic words (Свети, Преподобни, Saint, etc.) from keyword matching
- Single bio per day (Serbian crkvenikalendar.com): show on first non-moveable feast only
- Multiple bios (EN/RU): keyword match with used-bio tracking to prevent duplicates
- `Feast.description` takes priority over bio text
- Moveable feasts (`feast.moveable == true`) never get bios

### Haptics
`Haptics` enum is `@MainActor` with static `let` generators. Methods fire synchronously (no `DispatchQueue.main.async`) and re-prime after each fire to keep the Taptic Engine warm. All call sites are SwiftUI gesture handlers already on MainActor.

### Fasting Legend
`FastingLegend` reads locale from `LocalizationManager` to show translated labels (Oil/Fish/Strict/Feast, Уље/Риба/Строги/Празник, Масло/Рыба/Строгий/Праздник).

### Language Switching
Critical SwiftUI gotcha: `LazyVStack` caches cells by ID. Fix: `.id(viewModel.loadedLocale)` forces complete rebuild on locale change.

### JSON Compaction
Bundle files are compacted: null fields stripped, whitespace removed, `abbrev`/`icon`/`isFastFreeWeek` removed when empty. Models use optionals with defaults.

## Calendar Data (28 files)

`calendar_{locale}_{year}.json` — 4 locales × 7 years (2024-2030). Each file has 365/366 days with:
- `feasts[]` — saints and feast entries with importance, displayRole, type, moveable flag
- `readings[]` — scripture readings with title, reference, text, zachalo, service
- `fasting{}` — type, label, explanation
- `saintBios[]` — title + text (optional, stripped to null when empty)
- `greatFeast` — canonical feast ID or null
- `paschaDistance` — days from Pascha (negative=before, 0=Pascha, positive=after)

All files must be listed individually in `project.yml` resources section.

## Cloudflare Worker API

URL: `https://orthodox-calendar-api.ludikure.workers.dev`

| Endpoint | Response |
|----------|----------|
| `GET /api/health` | `{"status":"ok"}` |
| `GET /api/years` | `{"years":[2024,...,2030]}` |
| `GET /api/{locale}/{year}` | Full year calendar JSON from R2 |
| `GET /api/{locale}/{year}/{month}` | Single month filtered from year |

Worker source: `worker/src/index.ts`. R2 bucket: `orthodox-calendar-data`.

## Common Pitfalls

1. **Bundle resources**: Files in `data/output/` are NOT in the app bundle. Must copy to `OrthodoxCalendar/Localization/` AND list in `project.yml`.
2. **Zachalo numbers**: Differ between OCA, Serbian, and Russian traditions. Match by chapter:verse, never zachalo.
3. **Julian dates**: Fixed feasts use Julian calendar. Gregorian = Julian + 13 (for 1900-2099).
4. **Pascha distance**: Negative = before, 0 = Pascha day, positive = after.
5. **ForEach identity**: Never use gregorianDate alone as SwiftUI ID when locale can change.
6. **Feast descriptions**: Stored in `feast_descriptions.json` (not inline Python) to avoid Cyrillic quote escaping issues.
7. **New Calendar (en_nc)**: Fixed feasts on Gregorian dates, moveable feasts same as Old Calendar. `is_great_feast()` from Paschalion only used for moveable great feasts.
8. **R2 sync**: After rebuilding data, upload to R2 with `wrangler r2 object put --remote`.
9. **Fasting validation**: SPC fasting validated against pravoslavno.rs (93% match). Remaining gaps are individual saint-day upgrades.
